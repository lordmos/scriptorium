# {{書名}} 監査ログ

## ログフォーマット説明

各レコードには以下のフィールドが含まれます：

| フィールド | 説明 |
|------|------|
| 日付 | 操作日、形式 YYYY-MM-DD |
| Phase | 現在のプロジェクトフェーズ（Phase 1〜7） |
| 操作タイプ | 下記「操作タイプの分類」を参照 |
| 影響範囲 | どのファイル・章が影響を受けるか |
| 決定 | どのような決定をしたか |
| 理由 | なぜその決定をしたか |
| 実行者 | どのAgentまたは人が操作を実行したか |

### 操作タイプの分類

<!-- タグで分類することで、後から絞り込みやすくなります -->

| タグ | 意味 | 例 |
|------|------|------|
| `[结构]` | 章構成の変更 | 2章の統合、章の追加、順序調整 |
| `[术语]` | 用語関連の決定 | 翻訳の確定、用語の変更 |
| `[比喻]` | 比喩関連の決定 | 比喩の追加、比喩の廃止 |
| `[审查]` | レビューでの重要な発見 | 事実誤り・論理問題の発見 |
| `[源码]` | ソースコード関連の変更 | バージョン更新、重要な新ファイルの発見 |
| `[风格]` | 文体の調整 | 口調の変更、詳細度の調整 |
| `[读者]` | 読者フィードバックによる変更 | 難易度調整、説明の追加 |
| `[修复]` | エラーの修正 | コードエラー、事実誤りの修正 |
